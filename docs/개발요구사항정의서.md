## 1. 프로젝트 개요

### 1.1 프로젝트명
- **한국어**: 퍼스널 컬러 테스트
- **영문**: Personal Color Test
- **베트남어**: Trac nghiem Mau Sac Ca Nhan

### 1.2 프로젝트 목적
베트남 사용자를 대상으로 개인 퍼스널 컬러(사계절 유형)를 진단하는 무료 웹 애플리케이션을 제공한다. AI 사진 분석 또는 12문항 퀴즈를 통해 사용자의 피부톤, 눈 색상, 머리카락 색상을 분석하여 봄(Xuan)/여름(Ha)/가을(Thu)/겨울(Dong) 4시즌 중 가장 적합한 컬러 시즌을 판정하고, 맞춤형 메이크업/의류/헤어/장신구/제품 추천을 제공한다.

### 1.3 타겟 사용자
| 항목 | 상세 |
|------|------|
| **주요 타겟** | 베트남 여성 18~35세 |
| **관심사** | 뷰티, 메이크업, 패션, K-beauty, K-pop |
| **사용 환경** | 모바일 위주 (Samsung Internet, Chrome on Android, Safari on iOS) |
| **언어** | 베트남어 (vi_VN) |
| **유입 경로** | Facebook 그룹, 소셜 공유, 검색 엔진 |

### 1.4 핵심 가치 제안
1. **2트랙 진단**: AI 사진 분석(10초) 또는 질문형 퀴즈(3분) 중 선택 가능
2. **완전 무료**: 100% 무료, 사진 서버 전송 없음(프라이버시 보장)
3. **풍부한 결과**: 시즌 판정 + 3축 분석 차트 + 컬러 팔레트 + 메이크업/의류/헤어/장신구/연예인/제품 추천
4. **소셜 바이럴**: URL 해시 기반 결과 공유 및 Facebook 공유 지원
5. **교차 트래픽**: 자매 사이트(skintypetest.com)와 상호 링크

## 2. 시스템 아키텍처

### 2.1 아키텍처 유형
- **SPA (Single Page Application)**: 서버 없는 완전한 클라이언트 사이드 애플리케이션
- **빌드 도구 없음**: HTML/CSS/Vanilla JavaScript 단일 파일 구조, 번들러나 프레임워크 미사용
- **정적 호스팅**: Cloudflare Pages에서 정적 파일 직접 서빙

### 2.4 화면 전환 메커니즘
- CSS 클래스 기반 전환: `.screen` (비활성, `display:none`) / `.screen.active` (활성, `display:block`)
- `showScreen(id)` 함수가 모든 스크린에서 `.active` 제거 후 타겟 스크린에 추가
- 전환 시 `fadeUp` 애니메이션 적용 (0.45초, ease 커브, `translateY(24px)` -> `translateY(0)`)
- 전환 시 `window.scrollTo({ top: 0, behavior: 'smooth' })` 실행

### FR-02: AI 사진 분석

| 항목 | 상세 |
|------|------|
| **화면 ID** | `#ai-screen` |
| **설명** | 카메라 촬영 또는 사진 업로드를 통해 피부/머리/눈 색상을 추출하여 퍼스널 컬러 판정 |

#### FR-02-01: 카메라 뷰
- `navigator.mediaDevices.getUserMedia` API 사용
- 기본 `facingMode: 'user'` (전면 카메라)
- 해상도 목표: `640x854` (3:4 비율)
- 전면 카메라 시 CSS `scaleX(-1)` 미러링
- 얼굴 가이드: 점선 타원(`55% x 60%`) + 중앙 십자 표시 + "Dat khuon mat vao khung" 안내

#### FR-02-02: 카메라 전환
- 디바이스에 2개 이상 비디오 입력이 감지되면 카메라 전환 버튼 표시
- `facingMode`를 `user` <-> `environment` 토글
- `switchCamera()` 함수로 기존 스트림 중지 후 새 스트림 시작

#### FR-02-03: 사진 촬영 (`capturePhoto()`)
1. `<canvas>`에 비디오 프레임 캡처
2. 전면 카메라일 경우 좌우 반전(`translate` + `scale(-1,1)`) 적용
3. `getImageData()` → `capturedImageData`에 저장
4. `toDataURL('image/jpeg', 0.9)` → `<img>` 미리보기 표시
5. 카메라 스트림 중지, 촬영/분석 버튼 표시

#### FR-02-04: 사진 업로드 (`handleUpload()`)
1. `<input type="file" accept="image/*">` 사용
2. `FileReader.readAsDataURL()` → `Image` 객체 로드
3. 최대 800px으로 리사이즈 (비율 유지)
4. Canvas에 그린 후 `getImageData()` 추출
5. 카메라 불가 시 폴백 업로드도 별도 제공

#### FR-02-05: 스캔 애니메이션
- 분석 시작 시 `scan-overlay` 표시
- gradient 라인이 상단(15%)에서 하단(80%)으로 2초 주기 반복 이동
- `gold`/`rose` 그라데이션 + box-shadow 글로우 효과

#### FR-02-06: 색상 추출 엔진 (`extractFaceColors()`)
- **얼굴 영역 추정**: 이미지 중앙 기준 55% 폭, 55% 높이, 수직 42% 지점
- **피부(skin)**: 좌/우 볼 2개 영역 샘플링 후 평균
  - 좌측 볼: 중심에서 좌측 28%, 하단 5%, 크기 12%x12%
  - 우측 볼: 중심에서 우측 16%, 하단 5%, 크기 12%x12%
- **머리카락(hair)**: 이마 상단 영역, 중심에서 상단 42%, 크기 30%x10%
- **눈(eye)**: 좌/우 눈 2개 영역 샘플링 후 평균
  - 좌측 눈: 중심에서 좌측 18%, 상단 8%, 크기 10%x6%
  - 우측 눈: 중심에서 우측 8%, 상단 8%, 크기 10%x6%
- **노이즈 필터**: 밝기 25 미만 또는 245 초과 픽셀 제외

#### FR-02-07: 감지 색상 표시
- 분석 완료 후 피부/머리/눈 3개 색상 원형 프리뷰 표시
- 1.5초 후 결과 화면으로 전환

#### FR-02-08: 카메라 불가 폴백
- 카메라 접근 실패 시 업로드 폴백 UI 표시
- "Hoac lam trac nghiem" 텍스트 버튼으로 퀴즈 트랙 전환 가능

### FR-05: 결과 화면

| 항목 | 상세 |
|------|------|
| **화면 ID** | `#result-screen` |
| **설명** | 판정된 시즌의 종합 결과를 카드 형태로 표시 |

#### FR-05-01: 결과 헤더
- "Ket qua cua ban" 뱃지 (시즌 gradient)
- 시즌 이모지 (72px): 봄(🌸), 여름(🌊), 가을(🍂), 겨울(❄️)
- 시즌 이름 (gradient 텍스트): 예) "Mua Xuan Am Ap" / "Warm Spring"
- 시즌 서브타이틀: 예) "Tuoi sang -- Am ap -- Rang ro"
- 시즌 설명 텍스트

#### FR-05-02: 시즌 테마 적용
- `determineSeason()` 실행 후 CSS 커스텀 속성 동적 변경:
  - `--season-primary`: 시즌 메인 색상
  - `--season-light`: 시즌 라이트 배경색
  - `--season-gradient`: 시즌 그래디언트
- 결과 헤더 배경: `season-light` 색상 적용

#### FR-05-03: 3축 바 차트 (axis-card)
- 3개 축: Cool/Warm (temp), Dark/Light (depth), Soft/Clear (clarity)
- 양극성 가로 바: 중앙(50%) 기준 좌/우로 바 확장
- 정규화 공식: `pct = 50 + (value / maxPossible) * 50` (maxPossible = 15)
- 바 위치 결정:
  - `pct >= 50`: `left: 50%`, `width: (pct - 50)%` (우측으로 확장)
  - `pct < 50`: `left: pct%`, `width: (50 - pct)%` (좌측으로 확장)

## 5. 데이터 설계

### 5.1 점수 체계: 3축 양극성 시스템

```
                   음수(-) ←────── 0 ──────→ 양수(+)
  temp (색온도)   :   Cool(쿨톤)         Warm(웜톤)
  depth (명도)    :   Dark(어두움)        Light(밝음)
  clarity (채도)  :   Soft(소프트)        Clear(선명)
```

#### 5.1.1 점수 범위
- 퀴즈 모드: 각 축 -2 ~ +2 범위의 정수 (12문항 누적, 이론상 -24 ~ +24)
- AI 모드: `analyzeColorsToScores()` 결과, -12 ~ +12 범위로 클램핑

#### 5.1.2 각 축의 의미
| 축 | 음수 방향 | 양수 방향 | 주요 판별 요소 |
|----|-----------|-----------|---------------|
| **temp** | Cool (쿨톤): 파란/핑크 언더톤 | Warm (웜톤): 노란/피치 언더톤 | 피부 색조(Hue), R/B 비율, G/B 비율, 머리카락 색조 |
| **depth** | Dark (어두움): 깊고 진한 톤 | Light (밝음): 밝고 연한 톤 | 전체 명도 가중 평균 (피부 45% + 머리 35% + 눈 20%) |
| **clarity** | Soft (소프트): 낮은 대비, 부드러움 | Clear (선명): 높은 대비, 뚜렷함 | 피부-머리 명도 대비, 피부 채도, 눈-피부 대비, 전체 채도 |

### 5.2 시즌 친화도 공식

```javascript
const affinities = {
  spring: temp + depth + clarity,     // Warm + Light + Clear
  summer: -temp + depth - clarity,    // Cool + Light + Soft
  autumn: temp - depth - clarity,     // Warm + Dark + Soft
  winter: -temp - depth + clarity     // Cool + Dark + Clear
};
```

| 시즌 | temp | depth | clarity | 특징 |
|------|------|-------|---------|------|
| **봄 (Spring)** | +Warm | +Light | +Clear | 따뜻하고 밝고 선명 |
| **여름 (Summer)** | +Cool | +Light | +Soft | 시원하고 밝고 부드러움 |
| **가을 (Autumn)** | +Warm | +Dark | +Soft | 따뜻하고 깊고 부드러움 |
| **겨울 (Winter)** | +Cool | +Dark | +Clear | 시원하고 깊고 선명 |

판정 방식: 4개 시즌 친화도 중 최대값을 가진 시즌 선택 (`bestScore`)

### 5.3 AI 색상 분석 알고리즘 상세

#### 5.3.1 색상 추출 파이프라인
```
이미지 캡처 → 얼굴 영역 추정 → 영역별 픽셀 샘플링 → 노이즈 필터
→ RGB 평균 계산 → RGB → HSL 변환 → 3축 점수 산출
```

#### 5.3.2 RGB → HSL 변환
- 표준 HSL 변환 공식 사용
- 출력: `{ h: 0~360, s: 0~1, l: 0~1 }`

#### 5.3.3 temp (색온도) 산출 로직
| 단계 | 조건 | 점수 |
|------|------|------|
| 피부 Hue 분석 | Hue 18~55 (노란/주황) | temp +3 |
| | Hue 10~18 | temp +1 |
| | Hue >=300 또는 <10 (핑크/블루) | temp -2 |
| R/B 비율 | ratio > 1.5 | temp +2 |
| | ratio > 1.25 | temp +1 |
| | ratio < 1.05 | temp -2 |
| | ratio < 1.15 | temp -1 |
| G/B 비율 | ratio > 1.15 | temp +1 |
| | ratio < 0.95 | temp -1 |
| 머리카락 Hue | Hue 15~45 (따뜻한 톤) | temp +1 |
| | Hue >=200 또는 <10 (차가운 톤) | temp -1 |

#### 5.3.4 depth (명도) 산출 로직
```javascript
const overallL = skinHSL.l * 0.45 + hairHSL.l * 0.35 + eyeHSL.l * 0.20;
```
| 조건 | 점수 |
|------|------|
| overallL > 0.55 | depth +5 |
| overallL > 0.48 | depth +3 |
| overallL > 0.42 | depth +1 |
| overallL > 0.35 | depth -1 |
| overallL > 0.28 | depth -3 |
| overallL <= 0.28 | depth -5 |

#### 5.3.5 clarity (채도/대비) 산출 로직
| 단계 | 조건 | 점수 |
|------|------|------|
| 피부-머리 명도 대비 | contrast > 0.55 | clarity +2 |
| | contrast > 0.4 | clarity +1 |
| | contrast < 0.15 | clarity -1 |
| 피부 채도 | saturation > 0.45 | clarity +1 |
| | saturation < 0.18 | clarity -1 |
| 눈-피부 대비 | contrast > 0.4 | clarity +1 |
| | contrast < 0.08 | clarity -1 |
| 전체 평균 채도 | avgSat < 0.12 | clarity -2 |
| | avgSat < 0.18 | clarity -1 |

#### 5.3.6 점수 클램핑
- AI 모드 최종 점수: `Math.max(-12, Math.min(12, score))`

| **4시즌 컬러** | 봄(Spring)/여름(Summer)/가을(Autumn)/겨울(Winter) 4가지 색상 유형 |
| **색온도 (Temp)** | Cool(쿨톤, 차가운 계열) vs Warm(웜톤, 따뜻한 계열) |
| **명도 (Depth)** | Dark(어둡고 깊은 톤) vs Light(밝고 연한 톤) |
| **채도/대비 (Clarity)** | Soft(부드러운, 낮은 대비) vs Clear(선명한, 높은 대비) |
| **SPA** | Single Page Application, 페이지 리로드 없이 화면 전환하는 웹 애플리케이션 |
| **HSL** | Hue(색조)-Saturation(채도)-Lightness(명도) 색상 모델 |
