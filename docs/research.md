## 2. 컬러 분석 과학/알고리즘

### 2-1. Munsell 색상 체계 (Hue/Value/Chroma)

**개요**: Albert H. Munsell이 만든 3차원 색상 체계. 현대 퍼스널컬러 분석의 기반.

- **Hue (색상)**: 10개 주요 색상 × 10단계 = 100개 색상 스텝
- **Value (명도)**: 0(검정) ~ 10(흰색)
- **Chroma (채도)**: 0(무채색)부터 바깥으로, 고정 최대값 없음

**퍼스널컬러 적용**: SciART 12시즌 시스템이 Munsell 좌표 기반. 60-70개 캘리브레이션된 드레이프를 D65 조명 하에 사용.

| 시즌 | Hue 편향 | Value 범위 | Chroma |
|------|----------|-----------|--------|
| Bright Spring | Warm | 중-고 | 고 |
| Light Spring | Warm | 고 | 저-중 |
| Warm Spring | Warm | 중 | 중 |
| Light Summer | Cool | 고 | 저 |
| Cool Summer | Cool | 중 | 저-중 |
| Soft Summer | Cool | 중 | 저 |
| Warm Autumn | Warm | 중-저 | 중 |
| Deep Autumn | Warm | 저 | 중-고 |
| Soft Autumn | Warm | 중 | 저 |
| Cool Winter | Cool | 가변 | 중-고 |
| Deep Winter | Cool | 저 | 고 |
| Bright Winter | Cool | 가변 | 고 |

**HSL 대비 장점**: Munsell은 **지각적으로 균일** — 같은 수치 차이가 같은 지각 차이를 의미. HSL은 색상 영역별 불균일.

### 2-2. CIELAB 색공간 & CIEDE2000 색차 공식

**CIELAB (L\*a\*b\*)**: CIE 1976 지각 균일 색공간
- **L\***: 밝기 (0=검정, 100=흰색)
- **a\***: 녹-적 축 (음수=녹, 양수=적)
- **b\***: 청-황 축 (음수=청, 양수=황)

**CIEDE2000**: 가장 진보된 색차 공식. CIELAB의 지각 비균일성 보정.

#### RGB → CIELAB 변환 (3단계)

```
1단계: sRGB 감마 선형화
  c/255 > 0.04045 → c_linear = ((c/255 + 0.055) / 1.055) ^ 2.4
  그 외 → c_linear = (c/255) / 12.92

2단계: Linear RGB → XYZ (D65 조명)
  X = 0.4125*R + 0.3576*G + 0.1804*B
  Y = 0.2127*R + 0.7152*G + 0.0722*B
  Z = 0.0193*R + 0.1192*G + 0.9503*B

3단계: XYZ → CIELAB
  D65 화이트포인트: Xn=0.95047, Yn=1.0, Zn=1.08883
  L* = 116 * f(Y/Yn) - 16
  a* = 500 * (f(X/Xn) - f(Y/Yn))
  b* = 200 * (f(Y/Yn) - f(Z/Zn))
```

**적용**: DeltaE00 < 2.0 = 사람이 거의 구별 불가, > 5.0 = 확실히 다른 색. 시즌 팔레트와 추출 색상 비교에 사용.

**장점**: CIELAB 대비 지각 예측 오류 ~20% 감소. **블루 영역 문제** 해결 (쿨톤 피부 분석에 중요). 무채색 영역 보정 (소프트/뮤트 시즌 판정에 필수).

JS 구현: [color-diff npm](https://github.com/markusn/color-diff), [ciede2000-color-matching](https://github.com/michel-leonard/ciede2000-color-matching)

### 2-3. ITA (Individual Typology Angle) — 피부톤 객관적 측정

**공식**:
```
ITA = arctan((L* - 50) / b*) × (180 / π)
```

| ITA 범위 | 분류 |
|----------|------|
| > 55° | Very Light |
| 41° ~ 55° | Light |
| 28° ~ 41° | Intermediate |
| 10° ~ 28° | Tan |
| -30° ~ 10° | Brown |
| < -30° | Dark |

**적용**: depth 축의 객관적 측정 대체. L\*과 b\*만 사용하므로 조명 변화에 비교적 견고.

```javascript
function calculateITA(L, b) {
  return Math.atan2(L - 50, b) * (180 / Math.PI);
}

function itaToDepthScore(ita) {
  if (ita > 55) return 3;   // Very Light
  if (ita > 41) return 2;   // Light
  if (ita > 28) return 1;   // Intermediate
  if (ita > 10) return 0;   // Tan
  if (ita > -30) return -2; // Brown
  return -3;                  // Dark
}
```

**장점**: Fitzpatrick (자가 보고) 대비 객관적이고 재현 가능. 연속 스케일로 경계 문제 회피.

### 2-4. Von Kries 색순응 모델 (조명 보정)

**목적**: 임의 조명 아래 찍은 사진의 색상을 D65(표준 주광)로 정규화.

**핵심 과정**:
```
1. XYZ → LMS (원추세포 응답 공간) 변환
2. 대각 스케일링: L, M, S 각각 독립적으로 보정
   L_adapted = L_source × (Ld/Ls)
   M_adapted = M_source × (Md/Ms)
   S_adapted = S_source × (Sd/Ss)
3. LMS → XYZ 역변환
```

**간단 RGB 구현**:
```javascript
function vonKriesCorrect(pixel, estimatedWhite, targetWhite) {
  return {
    r: pixel.r * (targetWhite.r / estimatedWhite.r),
    g: pixel.g * (targetWhite.g / estimatedWhite.g),
    b: pixel.b * (targetWhite.b / estimatedWhite.b)
  };
}
```

**Bradford 행렬**: Von Kries 원본보다 정확. 특히 단파장(블루) 적응이 우수 — 쿨톤 분석에 중요.

**현재 문제**: 우리 시스템은 조명 보정 없음. 같은 피부라도 조명에 따라 완전히 다른 시즌으로 판정 가능.

### 2-6. 한국식 톤 분석 (한국식 퍼스널컬러 진단)

**3축 평가** (우리 시스템과 일치):
1. **언더톤**: Warm(웜) ↔ Cool(쿨) — 피부의 황-핑크 스펙트럼
2. **명도**: Light(라이트) ↔ Deep(딥) — 전체 밝기
3. **청탁도**: Clear(클리어) ↔ Soft(소프트) — 채도와 대비 수준

**동아시아 피부 특수 고려사항**:
1. **높은 b\* 기준치**: 동아시아 피부는 유럽 대비 황색(b\*) 값이 높음 → warm/cool 임계값 다르게 캘리브레이션
2. **좁은 L\* 범위**: "라이트"와 "딥"의 차이가 5-8 L\* 단위에 불과
3. **청탁도(clarity) 중심**: 한국 분석가들은 warm/cool 경계가 미묘한 동아시아인에서 **clarity가 가장 판별력 있는 축**으로 봄

**캘리브레이션 예시**:
```javascript
// 한국 피부 기준 임계값 (근사)
// b* > 18 = warm (유럽: ~15)
// L* > 67 = Light, L* < 62 = Deep (한국 피부 기준)
// 피부-머리카락 L* 차이 > 15 + 높은 chroma = Clear
```

## 4. UX/바이럴 패턴

### 4-1. 최적 질문 수

| 퀴즈 유형 | 최적 질문 수 | 근거 |
|----------|------------|------|
| 성격/평가 퀴즈 | **7-13개** | 데이터 품질 vs 완료율 최적 |
| 분기 로직 퀴즈 | 7개 | 분기가 정보 수집 가속 |
| 마케팅 리드젠 | 7-12개 | 이탈 가속 전 피크 |

**현재 12문항**: 성격/평가 퀴즈 최적 범위의 상단. 적절함.

### 4-2. 이탈률 패턴

| 이탈 지점 | 원인 | 영향 |
|----------|------|------|
| **Q1** | 약한 훅, 불명확한 가치 제안 | 단일 질문 최대 이탈 |
| 이메일/개인정보 요청 | 프라이버시 우려 | **전체 최고 이탈 트리거** |
| **Q3-Q5** | 초기 호기심 소진, 복잡성 증가 | "커밋먼트 갭" 구간 |
| 중반 (12문항 중 Q6-8) | 피로감, 불명확한 진행 | 꾸준한 감소 |
| 복잡한/주관식 질문 | 평균 2-3배 긴 응답 시간 | 질문별 이탈 스파이크 |

**복리 효과** (핵심):
- 10문항 각 95% 유지 → 전체 완료율 **60%**
- 각 90% 유지 → **35%**
- 각 85% 유지 → **20%**

→ **질문별 유지율의 작은 개선이 최종 완료율에 엄청난 영향**

**Q1 최적화**: 가장 쉽고, 3초 이내 답변 가능, 개인적으로 관련 있는 질문. 현재 Q1 "자연 머리색" → 적절한 워밍업 질문.

---

### 4-3. 프로그레스 바

연구 결과가 **혼재**:

- **긍정**: 작업 완료율 증가, 지속적 행동 변화
- **부정**: 초반 느린 진행 표시 시 이탈률 **상승**, 일부 연구에서 효과 없음

**권장 디자인**:
- 페이지 **하단** 배치 (상단 아님)
- **시각적 스케일만** 사용 — 페이지 번호/퍼센트 회피
- "Question 5 of 12" 정도의 미니멀 표시가 안전

---

### 4-4. 공유 가능한 결과 카드 (바이럴 핵심)

**이미지가 있는 퀴즈는 공유 확률 40배 높음.** BuzzFeed 퀴즈 트래픽의 75%가 소셜 공유에서 발생.

**결과 카드 체크리스트**:
1. 캐릭터/타입 이름 크게 표시
2. 각 결과 타입별 시각적으로 구별되는 일러스트
3. 2-3개 긍정적 특성 설명
4. 결과 타입에 맞는 **컬러 코딩 배경**
5. 사이트 브랜딩/URL 포함 (유기적 트래픽 유도)
6. 공유 버튼은 **감정적 피크** 직후 배치 (결과 공개 직후)
7. **미리 작성된 공유 텍스트**

**OG 이미지 사양**: 1200×630px (1.91:1). PNG(텍스트 위주) or JPEG 80-90%(사진 위주). 최소 400px 너비.

**우리 프로젝트**: 12시즌 각각에 대한 **동적 1200×630 OG 이미지** 생성이 소셜 공유에 결정적.

---

### 4-5. 호환성/비교 기능 (Ktestone 바이럴 핵심)

**Ktestone 바이럴 루프 구조**:
1. 유저 퀴즈 → 성격 타입 결과
2. 결과 페이지: **"누군가와 비교하기"** 기능
3. 링크 공유 → 친구도 퀴즈 → 양쪽 결과 비교
4. **양쪽 모두 결과 공유** → 사이클 반복
5. UGC(TikTok 결과 녹화) → 추가 유기적 발견

**성공 요인**:
- 단순함 (2분 이내 완료)
- 사회적 비교 본능 활용
- 커플/친구 콘텐츠 생성
- 특정 해시태그로 UGC 집적

**우리 적용**: "친구와 시즌 비교하기" — Warm Spring과 Cool Winter 호환성, 함께 스타일링 팁 등. **바이럴 계수를 2배로.**

### 4-6. 모바일 퍼스트 디자인

**베트남 모바일 트래픽 ~60-65%**, 유저 기준 98.6%가 모바일 접속.

**필수 가이드라인**:
- 터치 타겟: 최소 **44×44px** (Apple HIG)
- 옵션 간 간격: 최소 8px
- **화면당 질문 1개** — 스크롤 금지

- 폴드 위: 질문 텍스트 + 첫 2-3개 답변 옵션
- 최소 16px 본문 텍스트
- 야외/밝은 조명 가독성 위한 고대비
- 이미지 지연 로딩
- 초기 렌더링 2초 이내 목표
